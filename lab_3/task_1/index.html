    Реализация парсера в виде web-приложения

    В рамках данной работы необходимо было реализовать парсер в формате web-приложения. Для этого был использован FastAPI, парсер вызывается по POST-запросу localhost:8001/parse:

    from fastapi import FastAPI, HTTPException
from parser_threading import parse_and_save

app = FastAPI()


@app.post("/parse")
def run_parser():
    try:
        result = parse_and_save()
        return {"message": "Parser ran successfully", "result": result}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

    В директорию парсера также добавлены файлы, обеспечивающие логику работы приложения.

    Создание Dockerfile

    Для работы приложения необходимо создать три контейнера: для базы данных, парсера и FastAPI-приложения. Для этого были созданы два Dockerfile (образ для базы данных указан напрямую в docker-compose.yml):

    Dockerfile для FastAPI-приложения 
    FROM python:3.12-slim

WORKDIR /app
COPY /web-programming/lab_3_S2/task_1/requirements.txt .

RUN apt-get update && apt-get install -y \
    build-essential \
    libpq-dev \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*

RUN pip install --no-cache -r requirements.txt

COPY /HackathonsManagementSystem/ .
    Dockerfile для парсера 
    FROM python:3.12-slim

WORKDIR /parser
COPY /web-programming/lab_3_S2/task_1/requirements.txt .
COPY /web-programming/lab_3_S2/task_1/parser/creds.json .
COPY /web-programming/lab_3_S2/task_1/parser/parser_naive.py .
COPY /web-programming/lab_3_S2/task_1/parser/parser_threading.py .

RUN apt-get update && apt-get install -y \
    build-essential \
    libpq-dev \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*

RUN pip install --no-cache -r requirements.txt

COPY /web-programming/lab_3_S2/task_1/parser/call_parser.py .

    Создание docker-compose.yml

    Для управления контейнерами был создан файл docker-compose.yml. Он включает в себя настройку базы данных, портов, корневой директории, переменных окружения и зависимостей.

    docker-compose.yml 
    services:
  db:
    image: postgres:16
    container_name: db
    restart: always
    volumes:
      - ./db_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_USER=dptgo
      - POSTGRES_PASSWORD=iloveweb
      - POSTGRES_DB=hackathons
    ports:
      - "5434:5432"

  web:
    build:
      context: /Users/bill/PycharmProjects
      dockerfile: web-programming/lab_3_S2/task_1/web/Dockerfile
    depends_on:
      - db
    environment:
      DATABASE_URL: postgresql://dptgo:iloveweb@db:5432/hackathons
    command: python -m uvicorn main:app --host 0.0.0.0 --port 8000
    ports:
      - "8000:8000"

  parser:
    build:
      context: /Users/bill/PycharmProjects
      dockerfile: web-programming/lab_3_S2/task_1/parser/Dockerfile
    depends_on:
      - db
    environment:
      DATABASE_URL: postgresql://dptgo:iloveweb@db:5432/hackathons
    command: python -m uvicorn call_parser:app --host 0.0.0.0 --port 8001
    ports:
      - "8001:8001"

  db_data:
